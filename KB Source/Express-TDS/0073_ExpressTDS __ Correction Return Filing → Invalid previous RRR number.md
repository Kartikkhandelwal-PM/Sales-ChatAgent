Correction Return Filing: Steps to Avoid 'Invalid Previous RRR Number'

Step 1: Prepare to Upload Correction Return

Follow these steps to upload the FVU file without encountering errors related to the Previous RRR Number.

      

Step 2: Locate the Return to Be Corrected

Identify the original return requiring corrections.

Click on the View option to open the return details.

      

Step 3: Start the Correction Process

Click the Start Correction button.

      
The software will display the original return data.

Make the necessary corrections, such as updating:

PAN details

Challan details

Deductee details

Save the changes after completing the corrections.

Step 4: Validate the Correction Return

Select the Validate & Upload option to validate the corrected return data.

Step 5: Generate FVU for the Correction Return

After validation, choose one of the following based on your filing method:

Generate FVU (IT Portal): For filing through the Income Tax portal.

Generate FVU (OLTAS): For filing through OLTAS.

      
The software will generate the File Validation Utility (FVU) file for the corrected return.

Step 6: E-File the Correction Return

Click the E-file button.

      
You will be redirected to either:

TRACES portal

Income Tax portal

Log in to the portal and upload the FVU file.

Note: The Original RRR Number and Previous RRR Number fields are automatically filled during this step to ensure no errors occur during submission.

      

Additional Tips for Handling the FVU File

Locate the FVU File Folder

Open the folder where the FVU file was saved.

This folder typically contains the TXT file of the return.

Verify RRR Numbers in the TXT File

Open the TXT file using a text editor (e.g., Notepad or Notepad++).

      
Locate the lines containing:

Original RRR Number

Previous RRR Number

      
Confirm the correctness of these numbers:

The RRR (Receipt Reference Number) is unique for each tax filing.

This structured process ensures the correction return filing is seamless and errors related to RRR numbers are avoided.
